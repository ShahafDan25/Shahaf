Data Science Journal

See Code   ◦   Visit Site   ◦   Jun. 2022
Having graduated in May '22, all of a sudden June became surprisingly broing. I spent my time looking for DS & ML jobs in the Boston and NY area, while running across some pretty interesting APIs.
This project is the epitome of what a BU student with a BA in Computer Science and some experience can do with some data.

This project is a fully delopyed website. The front end was done using react and is connected via a flask backend to various REST APIs (using Python).
Some of the APIs used in the project include from NBA ( Players , Actions), Spotify, Yahoo Finance, and more.

For every API used, there are two main files within the GitHub Open Source Code I share: retrieve.py and analyze.ipynb
Those files are documented and commented very carefully and in a user-friendly manner to allow more developers to learn from the use of those APIs and to ease the process of retrieving and analyzing data on other developers.

Some of the DS & ML methods I was hoping to demonstrate include the use of Neural Networks, Anomaly Detection (Isolation Forrests, Outlier Factors), Classification Methods (SVM, KNN), Predictions, and more.

The Market Manager

See Code   ◦   Mar. 2020
The Market is a monthly event held by the Las Positas College Student Government.

In collaboration with the county's food bank, basic groceries are served to the community throughout an entire day.
The event is organized by the Student Government but includes volunteers from various student organizations.

As a constant volunteer, I noticed the inventory, volunteer sign-ups, and registration form are managed poorly and ineffectively.

I designed a software to allow Student Government to manage the periodical event with more confidence.
In the software, volunteers could sign ahead of time, create their own account, and see when volunteers are needed as well as the times their friend volunteered.
Additionally, inventory and leftover groceries could now be tracked via a software with user friendly data analysis graphics instead of pen and paper.
The software would also generate PDF reports to be submitted to the county's food bank of proper grocery use (per their request) regarding which groceries' supply needs to be increased or decreased.

Lastly, attendees at the Market are requested to sign up per arrival, which was done on google form without proper management of the data.
The software allowed to sign up online ahead of time and simply provide their email address upon arrival, rather than a 5-minute long repetetive sign up process.

The software today is used officially by the school and is running on the its servers under IT supervision and maintenance.
